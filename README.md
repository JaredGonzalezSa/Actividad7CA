### Desarrollo del Mini-Proyecto

#### 1. Definición del problema a resolver
Los estudiantes universitarios a menudo tienen dificultades para hacer un seguimiento de sus asignaciones, fechas de entrega y carga de proyectos a lo largo del semestre, lo que lleva a entregas tardías, desorganización y estrés. Se necesita una plataforma centralizada y sencilla para gestionar esta carga académica.

#### 2. Funcionalidades principales
1. **Registro de actividades:** Capacidad de crear tareas con título, descripción y fecha de vencimiento.
2. **Asignación de prioridades:** Sistema para etiquetar tareas como Alta, Media o Baja prioridad.
3. **Visualización de progreso:** Un panel principal estilo dashboard que muestre el porcentaje de tareas completadas frente a las pendientes.
4. **Notificaciones:** Alertas o recordatorios visuales para las tareas con fechas de entrega próximas.

#### 3. Roles del sistema
* **Estudiante Usuario Estándar:** Puede crear, editar, eliminar, priorizar y marcar como completadas sus propias tareas.
* **Administrador:** Encargado del mantenimiento de la plataforma y soporte técnico (no tiene acceso a las tareas privadas de los estudiantes).

#### 4. Flujo de uso
1. El estudiante ingresa al sistema de gestión.
2. Accede al panel principal, donde visualiza una lista de sus tareas pendientes ordenadas por urgencia prioridad y fecha.
3. Hace clic en "Agregar Tarea", llena los datos solicitados y guarda.
4. Conforme avanza en sus estudios, marca la casilla de "Completado" en la tarea correspondiente, lo que actualiza su barra de progreso general en la plataforma.

###  Evento de Contingencia Fallback
**Situación simulada:** Durante el desarrollo de la actividad, ..... reportó una pérdida de conexión.
**Procedimiento aplicado:** Ante esta situación, aplicamos nuestro plan de fallback. El Moderador asumió temporalmente el rol de Notetaker para no detener el flujo de trabajo en el documento compartido. Nos comunicamos por un canal asíncrono secundario whatsap para confirmar su estado de conexión y redistribuimos su tarea pendiente a través de la actualización de los Issues.

### Conclusiones y Presentación Final

* **¿Cómo se organizó el equipo?:** Utilizamos este repositorio de GitHub como base centralizada para el trabajo escrito. Creamos Issues para dividir las 5 tareas del mini-proyecto y utilizamos un documento compartido inicial para generar la lluvia de ideas en tiempo real.
* **¿Cómo gestionamos la comunicación?:** A través de WHATSAPP y DISCORD manteniendo siempre una netiqueta profesional, cuidando el tono y la claridad en nuestros mensajes.
* **Dificultades enfrentadas:** La principal dificultad al trabajar de forma remota fue coordinar la redacción simultánea y adaptarnos rápidamente cuando ocurrió la pérdida de conexión simulada.
* **Estrategias de resolución:** Aplicamos el plan de contingencia inmediatamente y utilizamos la asignación de Issues en GitHub para saber exactamente qué tarea estaba pendiente y quién la estaba cubriendo, evitando duplicar esfuerzos y manteniendo la responsabilidad colectiva.
